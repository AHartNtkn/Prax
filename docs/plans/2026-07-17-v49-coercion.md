### Task 1: The primitive and its own world

**Files:** Create `src/Prax/Coerce.hs`, `test/Prax/CoerceSpec.hs`; `prax.cabal`.
NOTHING shipped changes in this task — the primitive lands with its own racket

**1a. The module** — the `Coercion` record exactly as specced; `coerce :: Coercion ->
(Desire, [Action])` built from the boundary table:

- threaten: conditions = `coTrigger ++ [Neq coVictim "Actor", Not threatPath…]`;
  outcomes = threat marker + the motive-belief deposit (name = the constructed
  `punishes-<sid>`) + the extorted mark `<Actor>.extorted.<V>.<sid>`.
- comply: `[Match threatPath…, Not compliedPath…]` → `coDemand ++ [Insert
  compliedPath…, Delete threatPath…]`.
- defy: as today's shape (defied marker, threat deletion).
- punish: `Or [threat, defied] : coPunishWhen` → `coPunishOuts`.
- the want: `Desire ("punishes-" ++ sid) (Want (Or [defied(PraxD), threat(PraxD)] :
  renamedKernel) coWeight)`.
- the rename: generalize `renameVictim` to `namespaceKernel :: String -> [Condition]
  -> [Condition]` — collect the kernel's free variables in first-appearance order
  (reuse `conditionVars`); victim → `PraxD`; each other var not in {`Owner`} →
  `PraxW`, `PraxW2`, …; renaming applied through every Condition constructor
  op-preservingly (pattern segments via the `tokens` round-trip like `renameVictim`;
  Eq/Cmp/Calc/Count/Subquery operands and binders renamed consistently — the binder
  and its uses move together).
- guards: id single-segment; the victim reserved set; `authoredVarClash`/
  `authoredPatClash` (forbidden = the mechanism's names: `Actor`, `E`, `coVictim`)
  over trigger/demand/punishWhen/punishOuts/kernel — all loud with contextual
  messages; the registration contract in the module haddock.

**1b. CoerceSpec** — a self-contained racket world (extorter `mob`, victim `vic`,
bystander optional; a `barn` fact; a favor-demand; a burn-punishment; a vengeance
kernel `[Match "burned.barn.PraxD"]`-shaped BUT AUTHORED with plain names, e.g.
`[Match "burned.barn.V"]` — the rename law does the lifting). Pins (RED-first,
each observed):

1. Property 1: standing threat — `wait` never strictly beats both comply and defy
   (comparative assertion over the three scores).
2. Property 3: comply, then re-threaten → the renewed threat extracts nothing
   (comply unavailable; scores show no re-buy).
3. Property 5 (vengeance): `pickAction` — mob CHOOSES threaten at depth 2 holding
   the constructed want; the same world minus the want → threaten not chosen (the
   base-less selfNext path pinned both directions).
4. Punish against a STANDING threat (no defiance) fires.
5. The rename law: the authored plain-var kernel compiles to `PraxW*`/`PraxD` forms
   (inspect the constructed Desire); an authored kernel WITH a Prax var errors
   loudly; two fresh quantifiers get distinct names (`PraxW`, `PraxW2`).
6. The reserved set: victim=`Actor` errors loudly (the closed hole); victim=`PraxV`
   errors; a legal victim passes.
