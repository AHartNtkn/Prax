# v49 — Coercion: the leverage skeleton becomes a primitive; blackmail becomes its instance

Last of the audit-queued rounds. The v48 panel's verdict, binding here: `shakedown`
hardcodes its demo's three content choices (debt demand, exposure punishment,
believer-count motivation) into the mechanism, and the general model — v30's approved
epistemics (a threat is a motive-belief deposit; credibility is self-motivation; the
victim prices consequences through its own wants) — is evidence-agnostic. Fork (b)
chosen: an evidence-OPTIONAL primitive, with blackmail a thin instance.

## The primitive

New module `Prax.Coerce`:

```haskell
coerce :: String        -- ^ id (single segment; scopes this coercion's facts)
       -> String        -- ^ the victim role variable
       -> CoPresence    -- ^ co-presence template
       -> [Condition]   -- ^ trigger: what makes threatening available
                        --   (evidence-belief for blackmail; [] for a racket)
       -> String        -- ^ demand label (the comply action's text)
       -> [Outcome]     -- ^ demand: what compliance does
       -> (String, [Condition], [Outcome])
                        -- ^ punishment: label, EXTRA availability conditions,
                        --   effect (the mechanism owns the core gate)
       -> ([Condition], Int)
                        -- ^ punitive kernel: what the extorter values about the
                        --   punished state, and the weight
       -> (Desire, [Action])
```

## The mechanism/content boundary (decided here, per v48 constraint 5)

| | MECHANISM (in `coerce`, non-negotiable) | CONTENT (authored per instance) |
|---|---|---|
| threaten | threat marker `threatened.<sid>.<E>.<V>`; the motive-belief deposit; the extorted mark `<E>.extorted.<V>.<sid>` (sid-scoped — evidence-free worlds leave marks too); copresence + `Neq V Actor` + not-already-threatened gates | the trigger conditions; the label suffix |
| comply | the COMPLIED MARKER `complied.<sid>.<E>.<V>` inserted and guarded (`Not complied…` — the demand-independent re-buy guard, v48 constraint 2); threat deletion | the demand outcomes; the label |
| defy | defied marker; threat deletion | — |
| punish | availability core `Or [standing threat, defied]` (v48 constraint 3 — stalling never becomes safe) | extra availability conditions; the effect outcomes; the label |
| the punitive want | CONSTRUCTED by the mechanism: name `punishes-<sid>`, conditions `Or [defied, threat] : kernel` — the deposit and the desire share one name and one trigger BY CONSTRUCTION (v48 constraints 1 and 3 satisfied structurally, not by documented contract) | the kernel (what the extorter values: believers-of-evidence for exposure; the burned barn for vengeance) and the weight |
| guards | id single-segment; splice guards (v40 shape, v48 constraint 4) over trigger/demand/punishment/kernel with the mechanism's bound names (`Actor`, `E`, the victim variable, `PraxD`, `PraxW`) forbidden | — |

Registration contract stated at the primitive: the returned `Desire` goes to
`setDesires` and the extorter's `charDesires` — same as today's shakedown, now
documented as the credibility requirement it is.

## The re-buy guard's semantics, decided

v30's guard was `Not (debtPath …)` — under which a SETTLED debt would re-enable
extraction. That re-extraction path was never designed, exercised, or pinned (v30's
own bank lists "repeat / serial extortion" as a future mechanic). The mechanism's
`complied` marker is PERMANENT per (sid, extorter, victim): once bought, that coercion
cannot re-extract — the conservative reading; the bank item for deliberate serial
extortion stays open and unprejudiced.

## Blackmail, the thin instance

`Prax.Blackmail.shakedown sid copresence pat price w` keeps its signature and becomes
a `coerce` call: victim = `pat`'s first variable (the v30 convention, with its
existing guards); trigger = the evidence belief; demand = `owe` (the debt price);
punishment = expose (extra conditions: the evidence belief, the co-present
non-believing Hearer; effect: the sourced-hearsay plant); kernel = believers of the
evidence (`PraxW`/`PraxD` machinery as today). Composition, not wrapper: `shakedown`
IS an instance definition with blackmail-specific content; nothing re-exports the old
internals.

## Exactness, honestly

NOT byte-identical, in one itemized class: the mechanism's `complied` marker changes
`comply`'s compiled shape (its guard swaps `Not debtPath → Not complied…`; its
outcomes gain the marker insert), so analysis rows and fact traces that see comply's
atoms move by exactly that delta — itemized, nothing else. The decision arithmetic is
unchanged at every pinned point: `BlackmailSpec`'s exact v30 scores (−63.84 / −71.84 /
−75.80; the −54.2 tie) must hold untouched, and Village's goldens must be
decision-identical (fact-trace deltas limited to the marker family). The extorted
mark's tail changes `pat → sid` — the plan verifies no shipped want prices the mark's
tail (subtree pricing is unaffected).

## Verification

- The full second application, end-to-end (the audit's test, executed): a
  protection-racket arc with NO evidence — empty trigger, favor demand, barn-burning
  punishment, vengeance kernel — driving threaten/comply AND threaten/defy/punish,
  AND punish-against-a-STANDING-threat (constraint 3's pin), AND a re-threat after
  compliance extracting nothing (constraint 2's pin).
- The blackmail instance: BlackmailSpec's arithmetic pins untouched; VillageSpec's
  shakedown arc pins hold (marker-delta itemized); the credibility structural claim
  pinned (the deposit's name and the registered desire's name are the same generated
  name — a fixture with a mis-registered desire is impossible to author, which IS the
  pin: the primitive owns both).
- Splice-guard pins per authored surface; id/victim convention guards; the
  `renameVictim` machinery reused for kernel patterns (its op-preserving contract).
- Pre-gate: the three-lens panel (mandated by v48's spec for this round) runs on THIS
  document before the user gate.

## Out of scope

Serial/repeat extortion (banked, v30's item, now with stated semantics to design
against). Counter-coercion, bluffing, threat expiry (v30's bank, untouched). Evidence
patterns with multiple secondary roles beyond the existing guard's reach.
