# v49 — Coercion: the leverage skeleton becomes a primitive; blackmail becomes its instance

Last of the audit-queued rounds. Fork (b) per the v48 panel: an evidence-OPTIONAL
coercion primitive; blackmail a thin instance. REWRITTEN after this round's own
three-lens pre-gate panel (`.superpowers/sdd/v49-spec-review-*.md`) — the first draft's
kernel variable contract was self-contradictory (Critical: the primitive could not
express its own flagship instance), and the panel's remaining findings are folded in
below with [S]/[D]/[C] citations.

## The primitive

New module `Prax.Coerce`, with a named record surface (house style — anonymous tuples
carried no authored meaning [D-3]):

```haskell
data Coercion = Coercion
  { coId          :: String       -- single segment; scopes this coercion's facts
  , coVictim      :: String       -- the victim role variable (see the reserved set)
  , coTrigger     :: [Condition]  -- what makes threatening available — INCLUDING how
                                  -- the victim is reached: co-presence for an
                                  -- in-person threat, or any other authored channel
                                  -- (co-presence moved from mechanism to content
                                  -- [D-2]: delivery is a modeling choice; a letter
                                  -- coerces in absentia). The trigger must BIND the
                                  -- victim; an unbound victim flags in typeCheck's
                                  -- ordinary unbound-variable net.
  , coDemandLabel :: String
  , coDemand      :: [Outcome]    -- what compliance does
  , coPunishLabel :: String
  , coPunishWhen  :: [Condition]  -- EXTRA punish availability (the core gate is
                                  -- mechanism-owned)
  , coPunishOuts  :: [Outcome]    -- what punishment does
  , coKernel      :: [Condition]  -- what the extorter VALUES about the punished
                                  -- state (believers-of-evidence; the burned barn)
  , coWeight      :: Int
  }

coerce :: Coercion -> (Desire, [Action])
```

## The kernel's variable law [S-1, the Critical, resolution (c)]

The author writes the kernel with NO Prax-namespaced variables (the v40 law holds
verbatim — the splice guards on every authored field forbid them). The kernel
references the victim by `coVictim`'s name and any fresh quantifier (a believer, a
witness) by a plain name. `coerce` then alpha-renames INTO the Prax namespace, op-
preservingly (the `renameVictim` contract, generalized): the victim → `PraxD`, and
every other author-introduced free variable in the kernel that is not a mechanism
interface name (`Owner`) → `PraxW`, `PraxW2`, …. Only the mechanism's post-rename
OUTPUT is Prax-namespaced; no author ever writes one; the v40 usability-win pin
(secondary variables named `W`/`D` are ordinary author names) is the standing
regression detector. [The panel proved the alternative compiler-door bypass unsound:
worlds call `coerce` from raw Haskell exactly as mechanism modules do, so a bypass
either cannot identify its callers or exempts everyone.]

## The mechanism/content boundary (decided here)

| | MECHANISM (in `coerce`) | CONTENT (per instance) |
|---|---|---|
| threaten | threat marker `threatened.<sid>.<E>.<V>`; the motive-belief deposit; the extorted mark `<E>.extorted.<V>.<sid>`; `Neq V Actor` + not-already-threatened | `coTrigger` (including delivery/co-presence [D-2]); label |
| comply | the COMPLIED MARKER `complied.<sid>.<E>.<V>` inserted and guarded; threat deletion | `coDemand`; label |
| defy | defied marker; threat deletion | — |
| punish | availability core `Or [standing threat, defied]` (stalling never safe) | `coPunishWhen`; `coPunishOuts`; label |
| punitive want | CONSTRUCTED: name `punishes-<sid>`; conditions `Or [defied, threat] : renamed kernel` — deposit and desire share one name and one trigger by construction | the kernel and weight |
| guards | id single-segment; the victim reserved set — `coVictim ∉ {Actor, E, Owner, Hearer, Prax*}`, loud [S-5: this also CLOSES a latent shipped gap — today's derive-and-filter lets a `pat` whose first variable is `Actor` produce an unsatisfiable `Neq Actor Actor` threaten, silently]; v40 splice guards on every authored field | — |

The extorted mark stays mechanism, deliberately [D-4]: threatening is mark-worthy in
every coercion fiction, and the mark is pure priceable output. Its tail is `sid`, not
the evidence pattern — a stated EXPRESSIVENESS change [C-3]: the mark records WHICH
coercion, no longer WHAT evidence; a trait pricing blackmail-over-murder differently
from blackmail-over-theft loses its hook. Verified: no shipped or test consumer needs
the content tail (the one full-path fixture prices having-extorted-the-victim, which
the subtree form expresses more faithfully — see fixtures below).

Registration contract, stated plainly [D-5]: the returned `Desire` must be registered
(`setDesires`) and held (`charDesires`) — the name-identity half of credibility is
structural (one generated name), but an OMITTED registration still yields a silently
non-credible threat; the contract is documented at the primitive and the instance.

## The re-buy guard's semantics — an affirmative model decision [D-1]

The `complied` marker is PERMANENT per (sid, extorter, victim): one purchase per
coercion, ever. Authority, cited precisely: v48 constraint 2 (a demand-independent
marker) and the LEDGER's v30 bank item "repeat / serial extortion" (LEDGER:251,
2191-2196 — which explicitly names "a threat that renews" as the future mechanic).
Stated as the model claim it is: bought silence stays bought UNTIL serial extortion is
deliberately designed — renewable/expiring coercion (a monthly racket) is thereby
inexpressible for now, by decision rather than accident. [The panel's charge that the
bank citation was fabricated was itself checked and is wrong — the item lives in the
LEDGER's v30 row and bank section; the panel read only the v30 spec's §5. Both the
correction and the counter-correction are this record.]

## Blackmail, the thin instance

`shakedown sid copresence pat price w` keeps its signature, built as a `Coercion`:
victim from `pat`'s first variable (v30 convention + the new reserved-set guard);
trigger = evidence belief ++ `asRole victim copresence`; demand = `owe` (the debt);
punish = expose (extra: evidence belief, co-present non-believing Hearer; effect: the
sourced-hearsay plant); kernel = believers of the evidence, authored with PLAIN
variable names that `coerce` lifts into the namespace [S-1]. Composition, not wrapper.

## The contract: design properties, not the old design's decimals

[AMENDED at the gate — the user corrected a classification error: this round REPLACES
a design the audit judged unprincipled; it is not an implementation re-plumb, and
treating v30's compiled shapes and pinned decimals as ground truth would enshrine the
output of the thing being replaced. Compiled shapes are EXPECTED to differ; pins and
goldens re-capture wherever the re-founding moves them, each move explained in
MECHANICAL terms. The invariants are the PROPERTIES the v30 probe validated — the
decimals were only how the old pins happened to encode them.]

**The property contract (violating any of these is the BLOCK):**

1. **Stalling never dominates**: with a standing threat, waiting never strictly beats
   both comply and defy (the standing-threat punish gate's purpose).
2. **Audience scales fear** (blackmail instance): the compliance decision flips with
   onlooker count — comply dominates at two onlookers; defy/wait tie at one, with
   comply dominated. The COMPARATIVE ordering is the pin; the scores it happens to
   produce are recorded, not required.
3. **Repeat extraction is impossible**: after a purchase, a renewed threat extracts
   nothing (the permanent `complied` marker).
4. **Confession kills exposure leverage** (ConfessionSpec's arc): a confessed deed
   leaves the extorter nothing to expose — pinned as the property (exposure
   unavailable / wait chosen), not as its old decimals.
5. **Credibility is self-motivation**: threaten is chosen at depth 2 only by an
   extorter holding the punitive want — for BOTH kernels (exposure's base-bearing
   path and vengeance's base-less selfNext path [S-3]).
6. **The deterrence trait still deters**: a scrupulous extorter declines to threaten
   (the qualms fixture, re-pointed to the subtree mark — the more faithful semantics
   regardless).

**Expected drift, re-captured and itemized in mechanics terms**: any score decimals
that shift under the re-founded compilation (the recorded old values are the
comparison baseline for the itemization, nothing more); the two mark fixtures
[S-4, C-2]; whatever analysis rows the new compiled shapes produce (the completeness
lens PREDICTS near-zero movement [C-5] — that prediction is informational; drift is
itemized, not blocked, unless a property above breaks).

- v45 reserved families: coercion adds NONE, stated with the reason [C-4] — its
  marker families (`threatened`/`defied`/`complied`/`extorted`) are fiction-adjacent
  facts authors legitimately gate on (the qualms trait does), written with grounded
  values by ordinary actions; nothing about them is engine-owned.

## Verification

- **The racket arc lives in a spec-only fixture** (a `CoerceSpec` racket world) [C-6]
  — no shipped world gains content. End-to-end: empty-evidence trigger, favor demand,
  barn-burning punishment, vengeance kernel; threaten/comply; threaten/defy/punish;
  punish against a STANDING threat; a re-threat after compliance extracting nothing.
- **The vengeance self-motivation is PINNED via `pickAction`, not driven** [S-3]: the
  vengeance kernel pays base-less and selfNext-only (unlike exposure's
  existing-believer base), contingent on the predicted victim response — the racket
  extorter must CHOOSE threaten at depth 2, the exact analogue of BlackmailSpec's
  exposure pin, on the path where the two kernels diverge most.
- The kernel rename law pinned (plain-var kernel lifts to `PraxW*`; the v40
  usability-win pin green); the victim reserved-set guard pinned including the
  previously-latent `Actor` case; splice guards per authored field; registration
  contract documented.
- Docs task: Blackmail/Coerce haddocks, WALKTHROUGH's leverage prose, README's v30
  bullet annotated with the v49 re-founding [C-7].
