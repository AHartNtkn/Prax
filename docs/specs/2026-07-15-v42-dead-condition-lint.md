# v42 — Dead-condition lint: flag what the world can never satisfy

Third of the user-queued foundations passes, and the first NEW analysis written against
v41's unified surface. The defect class: an authored condition whose pattern can never
match anything the world can ever contain — a typo'd predicate segment, a renamed family
with a stale reader, a condition written for content that was later deleted. Today such
an action, function case, ForEach guard, or want just silently never fires; the checker
("Prax.TypeCheck", whose charter is __no false positives — the report is trustworthy__)
says nothing.

## The probe (live, all 7 shipped worlds — scratch `V42DeadProbe.hs`)

A prototype over the cooked tables found every shipped practice condition and want
clean, and produced exactly two classes of hits, both in feud's axiom bodies:

1. **□-lifted rule bodies** (`obliged.Obligor.wronged.X.Y`, …) — machine-generated by
   `liftObliged` (DEON property 1), speculative BY DESIGN: most worlds never obligate
   most rule contents. Not authored, not a bug.
2. **Authored kin-rule bodies** (`parent.P.X`, …) — `Prax.Kin.kinAxioms` is wired into
   feud WHOLESALE, and the fixture's own haddock documents the inert remainder as
   deliberate ("inclusion is free … harmless … no `parent.*` base fact exists until a
   wedding inserts one"). A rule library imported with some rules unfireable is a normal
   authoring pattern; an unfireable rule contributes nothing and harms nothing.

Both hits draw the same line: **axiom bodies are out of the lint's scope.** The
unambiguous-bug class is a dead condition at a site that makes an authored AFFORDANCE or
MOTIVE unreachable.

## Design

**One new check in `Prax.TypeCheck` (a world-model analysis: it reads the cooked
tables), one new producer-pool export from `Prax.Relevance`.**

- `Prax.Relevance` exports `producibleAtoms :: PraxState -> Maybe [[Sym]]` — everything
  the registered world can ever contain: the initial db's facts, every practice's
  insert-side atoms (`cookedOutcomeAtoms`' insert half over ALL practices — the drifter
  INCLUDED, unlike `worldAtomPools`: the lint asks "can this fact ever exist," and
  clock-moved facts exist), and every axiom head (`cookedRules`' `crHeads`, □-lifted
  forms included). `Nothing` = wild (an unresolvable `Call`): the lint goes silent —
  conservative, like every analysis on this surface. Axiom heads count as producers
  regardless of whether their rules can actually fire (a fireability fixpoint would be
  more precise; deliberately not built — conservativity here only ever silences the
  lint, never lies).
- New `TypeError` case `DeadCondition { teWhere :: String, teSentence :: String }`: at
  site `teWhere`, the positive pattern `teSentence` (rendered author-legibly via
  `symName`) may-unifies (`mayUnifySyms`) NO producible atom, so the condition can never
  hold and the site can never fire.
- **Sites scanned** (cooked, via `cookedDefs`/`cookedDesires`/`cookedWants`): action
  conditions, function-case conditions, `ForEach` guards (recursively, including inside
  function cases), desire wants, character wants.
- **Positions flagged**: positive `CMatch` conjuncts at conjunctive positions — top
  level and inside `CExists` (a dead positive there kills the `Exists`, killing the
  site). NOT flagged: negatives (`CNot`/`CAbsent` — a vacuously-true negation is
  plausible defensive authoring), `COr` clauses (a dead clause doesn't kill the Or;
  flagging reachable-but-partially-dead disjunctions is a different, weaker claim),
  `CSubquery` interiors (an always-empty set can be the intended meaning of a
  `Count`-≤-0 comparison), axiom bodies (the probe's finding, above).

The lint is exactly the v41 dividend: it is written once, against the cooked tables,
sharing `cookedOutcomeAtoms`/`cookedFnPool`/`mayUnifySyms` — no string-side walker
returns.

## Verification

- RED-first: a scratch-fixture world with a typo'd action conjunct (and one with a dead
  `ForEach` guard, one with a dead want) flags with the exact site and pattern; the same
  fixtures with the typo corrected are clean.
- The no-false-positives contract pinned on reality: all 7 shipped worlds remain
  lint-clean (`prax check` ×7 unchanged — the probe already showed this, the pin makes
  it durable); feud specifically pins that wholesale `kinAxioms` inclusion does NOT flag
  (the axiom-body exclusion, held by the real fixture that motivated it).
- Negative-position and Or-clause non-flagging each pinned (a `Not` over a never-produced
  family, a half-dead `Or`: both clean).
- Wildness pinned: a fixture with an undefined `Call` gets no `DeadCondition` (the
  dangling-`Call` check already reports the real problem).
- Goldens byte-identical; no engine, planner, or serialization change of any kind.

## Out of scope

The v43 hygiene bundle (queued next). A rule-fireability fixpoint (would let the lint
see through never-derivable axiom heads; precision without a demonstrated need).
Flagging inert axioms as a softer diagnostic class — would violate the checker's
no-false-positives charter on a documented-deliberate authoring pattern (feud).
