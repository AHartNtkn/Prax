# v51 — Lifting leaves the engine: the □-lift is content, and the gate dies

Last of the extension rounds (and of the whole audit arc). The assessment's charge:
v48 means-tested the lifting tax without moving the responsibility — `Prax.Derive`,
the GENERAL derivation engine, still contains deontic vocabulary (`obligedHead`),
the deontic transformer (`liftObliged`), and an always-lifting string path; the
engine still decides WHEN to lift (`deonticProducible`, a producer census recomputed
on every retable). v51 moves the □-lift to `Prax.Deontic`, gives `Derive` a general
transformer seam — and discovers at the probe that the v48 gate has no remaining
reason to exist.

## The insight: which logic your axioms close under is an authored decision

The paradigm (v44): engine responsibilities must be unrepresentable in world
content, and CONTENT responsibilities must not be engine magic. Closure under □ —
"obligations propagate through domain rules" (DEON property 1) — is a modeling
choice about a world's deontic logic. v48 made the ENGINE take it automatically,
means-tested by a producer census; that census forced the BUILD-ORDER INVARIANT
(states must be built setAxioms-outermost so the census sees the finished world) —
an order-sensitivity wart in every world's constructor. When the world DECLARES its
lift, the census has no job: only declaring worlds pay the doubled rule set (the
gate's whole purpose), and the declaration is order-free.

## The design

- **`Prax.Derive` becomes deontics-free.** `liftObliged`, `obligedHead`, and the
  Bool gate DIE here. `cookAxioms :: [Axiom] -> [CookedRule]` — no flag, no
  expansion: it cooks exactly the list it is given. `run`'s internal
  `mapMaybe liftObliged` dies; `run`/`closure`/`closureFrom`/`derived`/
  `contradiction` operate on exactly the axioms passed (general engine, general
  contract). The transformer seam is the AXIOM LIST ITSELF — see the setter.
- **`Prax.Deontic` owns its lift.** `obligedLift :: Axiom -> Maybe Axiom` — the
  moved `liftObliged`, verbatim semantics (all-Match bodies lift under
  `obliged.Obligor.`; anything else is Nothing), with the head literal drawn from
  the module's OWN vocabulary (`obligationPath`'s head — the `obligedHead`
  duplication dies; one home, and it is the module whose fiction it is).
- **The seam: `setAxioms :: [Axiom -> Maybe Axiom] -> [Axiom] -> PraxState ->
  PraxState`.** The setter applies the given transformers once and stores the
  EXPANDED list in `axioms`; `cookedRules` are computed here, once — not in
  `retable`, which loses its per-change `cookAxioms (deonticProducible st)`
  recompute (rules depend only on what the setter was given). A deontic world
  writes `setAxioms [obligedLift] axs`; every other world writes
  `setAxioms [] axs`. Transformers are call-site arguments, not state — nothing
  opaque parks in `PraxState`, no registration ordering exists, and the
  BUILD-ORDER INVARIANT dies with the census.
- **`Prax.Relevance.deonticProducible` DIES** — the census, its wild-Call
  conservatism, its db leg, and the v48 build-order comment block in `Engine`.
  Nothing means-tests the lift; the declaration IS the decision.

## What this preserves, exactly

- **Byte-identity everywhere** (residence/responsibility move of approved
  semantics — same classification as v50, stated up front): Village and Feud lift
  today (census true) and will declare `[obligedLift]` — the expanded rule set is
  the SAME set, so closures, scores, and transcripts are byte-identical. Every
  other world's census was false — no lifted cooked rules — and declares `[]` —
  no lifted rules: identical. The string path's always-lift asymmetry was inert
  for them (lifted rules fire only on `obliged.*` facts none can produce).
- **The ViewInvariant net keeps its v27 job, loses its v48 one, honestly**: the
  net's original purpose — the cooked hot path against the independent string
  reference — survives unchanged (`closure (axioms st)` over the now-expanded
  list). Its v48 gate-soundness role dies BECAUSE the divergence class it watched
  is now unconstructable: both paths consume the same expanded list by
  definition. The net's haddock is rewritten to say so.
- **DEON property 1 for declaring worlds** — now by declaration rather than
  detection, which is the property's honest shape: a world that never declares
  □-closure has a coherent weaker deontic logic (obligations that do not
  propagate), which is an authored model, not a silent failure — the engine does
  exactly what was declared, as with any un-set feature.

## What dies

`Derive.liftObliged`, `Derive.obligedHead`, `cookAxioms`'s Bool,
`run`'s internal lift, `Relevance.deonticProducible` (and every helper only it
consumed), `Engine`'s gate plumbing + the BUILD-ORDER INVARIANT comment block,
and the v48 gate pins (RelevanceSpec/DeriveSpec/ViewInvariantSpec census tests —
they pinned the census, which no longer exists; each deletion itemized).

## Exactness

Fiction transcripts, scores, AnalysisTable decision fields: byte-identical (the
same rule sets reach the same closures). Moves, itemized: the census pins die;
DeriveSpec's lift pins re-point to `Deontic.obligedLift` (same assertions, moved
subject); every `setAxioms` call site gains its transformer argument (Village and
Feud: `[obligedLift]`; all others `[]`); any test constructing states via
`setAxioms` re-plumbs mechanically. `axioms st` now holds the expanded list —
consumers enumerated at plan time (typeCheck's axiom walks see lifted forms:
`Obligor` is body-bound in every lifted rule, so no new UnboundVar; the reserved
scan sees `obliged.*` heads, unreserved). Persist unaffected (axioms are code,
never serialized — v50-confirmed).

## Verification

- Byte-identity: full suite green with NO fiction golden moved (FeudSpec/
  VillageSpec deontic arcs — comply→owe→oblige, the feud pins — unchanged).
- `Deontic.obligedLift` pins: the moved lift's law (all-Match lifts, mixed-body
  refuses) re-pinned at its new home; the lifted-form shape pin (obliged.Obligor
  prefix on body and heads).
- The seam: `setAxioms [obligedLift]` produces the lifted cooked rules
  (DeriveSpec's v48 expansion pin, re-pointed); `setAxioms []` produces none; a
  world built in ANY setter order gets identical rules (the build-order wart's
  death, pinned positively).
- ViewInvariant: net green over declaring and non-declaring worlds; haddock
  honesty check (no gate-soundness claim survives).
- Deaths grep-proof: `liftObliged|obligedHead|deonticProducible` zero in src/.
- Pre-gate: the three-lens panel runs on this document (engine-surface round).
